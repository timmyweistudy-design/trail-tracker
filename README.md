# 步道誌 Trail Tracker

一款搜尋登山步道與親子步道的**跨平台手機 App**（手機瀏覽器 / PWA，iOS / Android 皆可使用），提供步道分級，並記錄使用者的步行路徑、計算里程、步數與消耗卡路里。步道資料以**政府開放資料**為主要來源。

> ✅ **MVP 已可執行**（手機優先 PWA 網頁版）。本文件含大綱與執行方式。

## 🚀 快速開始

```bash
./run.sh              # 啟動，瀏覽器開 http://localhost:8765
./run.sh --refresh    # 先更新政府開放資料再啟動
```

> 手機測試：手機與電腦連同一 Wi-Fi，瀏覽器開 `http://<電腦IP>:8765`，可「加入主畫面」當 App 用。
> 電腦預覽路徑記錄：勾選頁面下方「模擬模式」即可（電腦無 GPS）。

## ☁️ 部署（Render 靜態站）

本專案是純前端 PWA，用 Render Static Site 部署，設定已寫在 `render.yaml`。

1. 把專案推到 GitHub（已先在本地 `git init` 並 commit）：
   ```bash
   git remote add origin git@github.com:<帳號>/trail-tracker.git
   git push -u origin main
   ```
2. Render → **New → Blueprint** → 選此 repo，會自動依 `render.yaml` 建立靜態站
   （或 **New → Static Site**，手動設定 Publish directory = `web`，無 build command）。
3. 完成後即取得 `https://trail-tracker.onrender.com` 之類網址，手機可直接開、加入主畫面。

> ⚠️ 定位（GPS）需在 **HTTPS** 下才能用 — Render 自動配 HTTPS，沒問題；本地 `localhost` 也算安全來源。

### 目前實作
- **探索**：搜尋 + 篩選（親子友善 / 難度 / 地區），步道卡片與詳情頁（地圖、海拔、路面、官方連結）
- **周邊美食**：開啟步道詳情時，即時向 OpenStreetMap 查詢附近 4 公里餐飲（餐廳/咖啡/烘焙），依距離排序、localStorage 快取
- **記錄**：GPS 即時軌跡、距離、步數、卡路里、時間、配速；可暫停 / 結束
- **我的**：個人資料（體重身高，用於步數卡路里估算）、行程歷史
- **資料（全台 ~900 條步道）**：
  - 林業署 117 條（含難度分級、海拔、路面，59 條親子友善）
  - OSM 社群 ~784 條全台具名步道（爬蟲擴充覆蓋；多為未分級的郊山/古道）
  - 座標統一轉 WGS84，跨來源依「同名 + 座標相近」去重
- PWA：可離線開啟、加入主畫面

## 一、核心功能

### 1. 步道搜尋
- 搜尋登山步道與親子步道
- 篩選條件（暫定）：地區、難度、長度、是否適合親子、是否有無障礙設施
- 步道資訊頁：路線地圖、海拔變化、預估時間、注意事項

### 2. 步道分級
- 為每條步道評定難度等級（暫定分級維度，細節再議）：
  - 距離長度
  - 累積爬升 / 海拔落差
  - 路面狀況與坡度
  - 親子友善程度
- 以視覺化標籤呈現（例如：入門 / 一般 / 進階 / 挑戰）

### 3. 路徑記錄
- 即時 GPS 軌跡記錄
- 開始 / 暫停 / 結束 行程
- 軌跡儲存與歷史回顧

### 4. 運動數據計算
- **公里數**：依 GPS 軌跡計算總距離
- **步數**：整合裝置計步器 / 加速度感測器
- **卡路里**：依使用者體重、距離、海拔、時間等估算消耗

### 政府開放資料候選來源（台灣）
- 政府資料開放平臺（data.gov.tw）— 步道、登山路線相關資料集

- 各縣市政府開放資料（觀光、步道）

> 詳細來源盤點與實測欄位見 [`docs/data-sources.md`](docs/data-sources.md)。
> 已驗證林業署「步道基本資料」API 可用（117 條，含難度分級 `TR_DIF_CLASS`、長度、海拔、入口座標等欄位）。

## 四、技術初步構想

由於需 iOS / Android 雙平台，採**跨平台框架**開發較有效率。

| 項目 | 建議方案 | 備註 |
| --- | --- | --- |
| 前端 / App | **Flutter**（首選）或 React Native | 一套程式碼跑雙平台 |
| 後端 | Python（FastAPI）或 Node.js | 負責整合政府開放資料、提供 API |
| 資料庫 | PostgreSQL（+ PostGIS 處理地理資料） | 適合儲存步道路線與空間查詢 |
| 地圖 | Mapbox 或 OpenStreetMap | 開源 / 成本較低 |
| 定位 / 感測 | 裝置 GPS、計步感測器 | 路徑記錄與步數 |

> 資料流程：定期擷取政府開放資料 → 清洗 / 統一格式 → 存入資料庫 → App 透過後端 API 取用。
